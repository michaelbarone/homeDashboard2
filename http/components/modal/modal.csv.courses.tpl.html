CSV format explained for term by term Course data

All data is only stored within this browser session. No data will be sent or stored on any server for this application.
All Emplid fields are hashed when imported to help anonymise both student and instructor identities.

CSV example:

Emplid Term Code Term Description Course Class Title Official Grade Instructor Emplid Department Status Desc
783089809 2175 Spring 2017 MATH 30 Calculus I A 986085804 Mathematics Enrolled
783089809 2178 Fall 2017 MATH 31 Calculus II A 986085804 Mathematics Enrolled






Emplid The student ID or any unique key to identify a student. This id can occur multiple times in the data set, and will be grouped as the same student.
Term Code A numeric code for each academic term where each term code is larger than the last
Term Description A human readable description of the term
Course The course short title which is made up of the department short code and the course number
Class Title The official title of the course
Official Grade The official grade earned by this student, in this course, this term
Instructor Emplid The instructor ID who taught this course
Department Full department name
Status Desc Enrollment status for this course, can include "Dropped" or "Enrolled"
